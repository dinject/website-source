About Kanuka

Uses APT

Kanuka performs DI very fast using compiled code avoiding reflection and classpath scanning.

Kanuka uses annotation processing to generate Java source code to perform dependency injection much like Google Dagger.

Annotate components with @Singleton and use @Inject to specify constructor injection or field injection.

```java @Singleton public class CoffeeMaker { private final Pump pump; @Inject CoffeeMaker(Pump pump) { this.pump = pump; } ... ```

 

 

@PostConstruct & @PreDestroy

Kanuka supports bean Lifecycle via @PostConstruct and @PreDestroy. These methods are most often used to open resources on startup (e.g. connections to remote services and databases) and close those resources on shutdown.

```java ... @PostConstruct void onStart() { // perform action on startup, open resources etc } @PreDestory void onStart() { // close resources on shutdown } ```

 

 

@Factory and @Bean

Sometimes we need to have some logic when creating beans. We can use @Factory on a class that has methods annotated with @Bean to perform logic when creating beans.

In Spring DI this equates to using @Configuration and @Bean.

```java @Factory public class MyFactory { private final Configuration configuraton; @Inject MyFactory(Configuration configuraton) { this.configuraton = configuraton; } @Bean Foo createFoo() { // potentially perform some logic return new Foo(...); } @Bean Bar createBar(Foo foo) { // potentially perform some logic return new Bar(...); } } ```

 

 

Testing

During testing we sometime want to wire the graph but supply some test doubles or mocks for some of the dependencies (like Spring test scope).

```java @Test public void myComponentTest() { // we have some test doubles we want to use MyRedisApi mockRedisApi = mock(MyRedisApi.class); MyDatabaseApi mockDb = mock(MyDatabaseApi.class); // create the context programmatically with some // test doubles rather than the real dependencies try (BeanContext context = new BootContext() .withBean(mockRedisApi) .withBean(mockDb) .load()) { // perform a component test (using the test doubles) CoffeeMaker coffeeMaker = context.getBean(CoffeeMaker.class); coffeeMaker.brew(); } } ```

 

 

Modules

We can also want perform dependency injection with beans in other modules (jars).

We do this by using @ContextModule to define the module dependencies. The dependsOn defines other modules that should be wired before this module and that provide beans that can injected in our module.

```java @ContextModule(name="coffee", dependsOn="common-configuration") ```

Getting started

Maven

Add the 2 dependencies.

```xml io.kanuka kanuka 0.3 io.kanuka kanuka-generator 0.3 provided ```

Use BeanContext

Use SystemContext or BootContext to obtain the bean context and use the beans.

```java // obtain a bean from the context and use it CoffeeMaker coffeeMaker = SystemContext.getBean(CoffeeMaker.class); coffeeMaker.brew(); ``` ```java // create the context programmatically // ... with some options try (BeanContext context = new BootContext() .withNoShutdownHook() .withBean(someTestDouble) .load()) { CoffeeMaker coffeeMaker = context.getBean(CoffeeMaker.class); coffeeMaker.brew(); } // Using try with resources block above such that // the context is closed ```
